Why Maintenance Agent
What makes it different

Most property tools are either too simple for professional managers or too complex for small landlords. We're building both, on the same platform.

→
One source of truth
Maintenance events, lease data, invoices, and legal cases all live in the same system and cross-reference each other. No more spreadsheet bridges between tools.
→
Swiss law built in
The legal engine isn't a PDF library — it evaluates actual Swiss law rules against live reference rates and building-specific data. Automatically flags and routes legal exposure.
→
All four parties, one platform
Manager, owner, tenant, and contractor each have a role-scoped interface. Communication flows through the platform, not via email chains and WhatsApp groups.
→
Financial intelligence, not just bookkeeping
Per-building financial snapshots, strategy profiles, and cashflow forecasting. The platform tells owners what their buildings are doing financially — not just what was paid.
→
Scales from 1 to many buildings
The same backend powers the professional manager view and a simplified private landlord mode (Phase 2). No separate product, no data migration needed.
→
Built for AI augmentation
Triage, classification, and legal analysis already use rule-based AI. The platform is architected to add LLM-based triage and self-service resolution as the next layer.

Product roadmap

Six phases taking the platform from production-ready to a full-coverage Swiss property intelligence layer. Click any phase to explore.

P0
Foundation Hardening
Now → Month 1
In progress ▾
Clear the critical path blockers before adding any new features. Three items directly block monetizable work.
Wire
Email Delivery (SMTP / SendGrid)
EmailOutbox model and dev sink built. Wiring production transport enables rent reminders, invoice delivery, and tenant notifications.
Wire
Legal DSL Variable Resolver ✓
LegalVariable, LegalVariableVersion and LegalRule wired to live SNB/BFS data. Canton-scoped reference interest rate ingestion complete.
Wire
Digital Lease Signing (Skribble)
SignatureRequest model exists. Wire to Skribble API — the Swiss-compliant e-signature provider. Table stakes for private landlord positioning.
Refactor
Service Layer Extraction
legal.ts, tenants.ts, leases.ts have business logic in route handlers. Extract to services before P1 financial work adds complexity.
P1
Cashflow Layer
Month 1–3
Planned ▾
Build the financial visibility layer for building owners — cash in, cash out, per-building balance — without crossing into accounting.
Build
QR-Invoice Rent Demand Generation
Generate ISO 20022 compliant Swiss QR-invoices for monthly rent demands from lease data. Send via email. Track payment status automatically.
Build
Bank Payment Reconciliation (MT940)
Accept MT940/CSV exports from major Swiss banks. Auto-match incoming payments to open rent demands by amount + reference.
Extend
Cashflow Service
Monthly snapshot of income, expenses, vacancy loss, and net balance per building. Populates BuildingFinancialSnapshot for owner reporting.
Build
Rent Arrears Workflow
Escalating reminders: friendly nudge (D+3) → formal notice (D+14) → Betreibungshinweis template (D+30). Fully automated email delivery.
Build
Nebenkosten Statements
Annual utility charge statement generation. Input: expense invoices tagged as Nebenkosten over 12 months. Output: PDF per tenant with settlement.
Extend
Owner Cashflow Dashboard
Per-building monthly cashflow view: income bar, expense bar, net balance trend. Real data from BuildingFinancialSnapshot.
P2
Private Landlord SaaS
Month 3–6
Planned ▾
The current product is built for professional managers. This phase creates a simplified entry point for private individuals owning 1–20 properties — same backend, different UX skin and onboarding.
Product
Self-Service Onboarding Wizard
Private landlord operational in under 15 minutes: add building → add unit → upload lease → set rent → generate first QR-invoice. Multi-language from day one.
Build
Simplified Landlord Mode
OrgConfig flag that collapses Manager and Owner personas into a single simplified interface for private landlords who manage their own properties.
Build
Multilingual Tenant Portal (FR/DE/IT)
All tenant-facing pages served in FR, DE, or IT based on tenant language preference. Without this, the platform is only usable by German-speaking tenants.
Extend
Canton Rent Notification Forms
Generate the correct canton-specific initial rent notification form from lease data, pre-filled and ready to sign. First target: Form-10/11 (Zurich, Bern).
Extend
Tenant PWA (Mobile App)
Wrap the tenant portal as a PWA — installable on iOS/Android, push notifications, offline document access. Replaces phone calls for issue reporting.
Build
Document Vault
Structured document storage — lease PDFs, signed addenda, Nebenkosten statements, correspondence — linked to tenant and unit with search.
P3
AI Triage Upgrade
Month 5–8
Planned ▾
Extend the existing triage service from classify-and-route to resolve-before-dispatch. The infrastructure is there. The intelligence layer is not.
Build
Media Attachments on Requests
Tenant attaches photos/video when submitting a request. Forwarded to contractor before dispatch — eliminates "I need to see it first" call-outs.
Build
LLM-Based Triage
Replace/augment rule-based triage with LLM classification. Input: free-text + photos. Output: category, urgency score, recommended action, confidence.
Extend
Self-Service Resolution Q&A
When tenant submits a request, present structured Q&A to resolve at intake without dispatch. Target: resolve ~20% of requests without contractor involvement.
Build
Maintenance Analytics
Requests by category/building, avg time-to-resolution, self-resolution rate, contractor performance, cost-per-category. Decision-ready reporting for managers.
P4
NOI Intelligence
Month 8–14
Planned ▾
Connect cashflow + maintenance analytics + vacancy data into a per-building P&L proxy. Explicitly not accounting — the output is decision intelligence, not a financial statement.
Build
Maintenance Cost Tagging
Link every closed maintenance request to its total cost. Tag as capex or opex. Maintenance events appear as tagged expenses in the financial model.
Extend
Vacancy Cost Tracking
Calculate and display vacancy cost as a negative cash event. Links to reason where known. Makes the cost of slow re-letting visible in the owner dashboard.
Product
NOI Dashboard
Per-building: gross income, total expenses (with maintenance breakdown), vacancy loss, net operating income proxy. Clear enough for an owner meeting.
Build
Predictive Maintenance Signals
Use AssetIntervention + DepreciationStandard to surface forward-looking signals: aging assets, recommended replacement budget, risk alerts.
Build
Accountant Export
One-click annual summary for accountant: all income, expense invoices, Nebenkosten statements, lease periods, vacancy periods — structured export.
Infra
Live Bank Feed (SIX bLink)
Replace MT940 manual import with live bank feed via SIX bLink. Automatic daily reconciliation. Built after MT940 is proven in production.
P5

Build
Energy Consumption Tracking
Per-building energy data ingestion and reporting. Cantonal compliance deadlines surfaced proactively.
Build
Retrofit ROI Modeling
Connect depreciation standards and energy data to model the financial return on insulation, heating, and window retrofits.
Product
Institutional Owner Reporting
