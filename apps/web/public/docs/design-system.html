✓ Decided / Reference ◆ Architecture
ARDesign System Architecture

Every visual decision in the app flows through a single source of truth chain: semantic tokens defined once in globals.css @theme {} feed Tailwind v4, which generates utility classes consumed by CVA-powered UI primitives in components/ui/, with domain status mapping handled by statusVariants.js. Format helpers in lib/format.js and layout helpers from lib/utils.js (cn()) complete the system. Click any layer to expand its detail.

AR03Golden Rules — quick reference
✓ One token file. All colors, surfaces, and borders live in globals.css @theme {}. Zero theme extensions in tailwind config.
✓ cn() always. Use cn(twMerge(clsx())) from lib/utils.js for every conditional or composed className.
✓ CVA for variants. Any component with 2+ visual states gets a CVA definition in components/ui/.
✓ statusVariants for domain status. Never define per-file color maps. Import the relevant mapper and pass to Badge.
✓ format.js for all formatting. formatChf, formatDate, formatDateTime — never inline new Intl.NumberFormat().
✓ No new .css files. Shared styles go in globals.css @layer components via @apply.
⚠ No inline style={{}}. Use Tailwind utilities or @apply classes. Inline styles bypass the token system.
⚠ No template-literal className. `bg-${x}-100` is invisible to Tailwind's scanner — Tailwind will not include that class in the build output.
